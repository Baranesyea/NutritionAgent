# אפיון מערכת — סוכן תזונה אישי

## סקירה כללית

מערכת סוכן תזונה אוטונומי שמתכננת, מנהלת ומתקנת את התזונה השבועית של ערן. המערכת מורכבת משלושה רכיבים:

1. **GitHub Repo** — מאגר המידע המרכזי (source of truth)
2. **GitHub Pages Dashboard** — ממשק ויזואלי לצפייה ואינטראקציה
3. **Claude Code Routines** — סוכנים אוטונומיים שרצים בזמנים קבועים

בנוסף, שיחה ייעודית בקלוד משמשת כערוץ תקשורת לדיווח סטיות ועדכונים, שנדחפים לריפו ומתעדכנים בדשבורד.

## רכיב 2 — GitHub Pages Dashboard

### תיאור כללי

עמוד HTML סטטי שיושב על GitHub Pages ומציג את כל המידע מקבצי ה-JSON בריפו. הוא קורא ישירות מה-raw files של הריפו (או דרך GitHub API).

### מסכים / סקשנים

#### 1. סקשן עליון — סטטוס שבועי

- שבוע נוכחי (תאריכים)
- פס התקדמות קלוריות שבועי (נאכל / נותר)
- פס חלבון שבועי
- סטטוס: "און טראק" / "סטייה קלה" / "צריך תיקון"
- ימים שנותרו בשבוע

#### 2. סקשן תפריט יומי

- כרטיסייה לכל יום בשבוע
- היום הנוכחי מודגש
- לכל ארוחה: שם, מרכיבים, מאקרוס
- סה"כ מאקרוס ליום
- סטטוס: מתוכנן / בוצע / סטייה

#### 3. סקשן רשימת קניות

- מחולקת לקטגוריות
- אפשרות לסמן פריטים כ"נקנה" (שמירה ב-localStorage)

#### 4. סקשן Meal Prep

- 2 סשנים בשבוע
- לכל סשן: רשימת הכנות, סדר, זמן משוער
- תאריך ושעה

#### 5. סקשן לוג

- רשימת אירועים כרונולוגית
- סטיות, תיקונים, עדכוני משקל, הערות
- כל ערך עם timestamp וסוג

#### 6. סקשן מאזן שבועי

- טבלה: יום / מתוכנן / בפועל / הפרש
- שורת סה"כ שבועי
- ויזואליזציה (בר צ'ארט פשוט)

### עיצוב

- מובייל פירסט (ערן ישתמש בזה מהנייד)
- כהה, נקי, מינימלי
- עברית RTL
- ללא פריימוורק — HTML/CSS/JS וניל
- קורא JSON ישירות מ-GitHub raw URLs

## רכיב 3 — Claude Code Routines

### רוטין 1: תכנון שבועי (Weekly Planner)

**טריגר:** Schedule — כל יום חמישי בשעה 07:00

**מה עושה:**
1. קורא את `profile.json` מהריפו
2. קורא את `history/` לייחוס (מה היה בשבועות קודמים, מה חוזר יותר מדי)
3. מייצר תפריט שבועי חדש ל-7 ימים — `menu.json`
4. מייצר רשימת קניות — `shopping-list.json`
5. מייצר סשני meal prep — `meal-prep.json`
6. מאפס את `balance.json` לשבוע חדש
7. מאפס את `log.json` לשבוע חדש
8. מעביר את השבוע הקודם ל-`history/`
9. דוחף הכל לריפו (commit + push)
10. שולח מייל לערן דרך קונקטור Gmail עם סיכום: תפריט, רשימת קניות, לינק לדשבורד
11. מכניס 2 אירועי meal prep ליומן דרך קונקטור Google Calendar

**פרומפט (תמצית):**
```
אתה סוכן התזונה של ערן. קרא את profile.json ובנה תפריט שבועי חדש.
כללים: [הפנייה ל-CLAUDE.md]
פלט: עדכן את menu.json, shopping-list.json, meal-prep.json, balance.json, log.json.
דחוף לריפו, שלח מייל, הכנס אירועים ליומן.
```

### רוטין 2: צ'ק-אין יומי (Daily Check-in)

**טריגר:** Schedule — כל יום בשעה 08:00

**מה עושה:**
1. קורא את `menu.json` — מה מתוכנן להיום
2. קורא את `log.json` — האם יש דיווחי סטייה מאתמול שלא טופלו
3. קורא את `balance.json` — מה המצב השבועי
4. אם יש סטייה שלא טופלה:
   - מחשב תיקון
   - מעדכן את `menu.json` לימים שנותרו
   - מעדכן את `balance.json`
   - כותב ל-`log.json` שבוצע תיקון
   - דוחף לריפו
5. שולח מייל לערן:
   - ארוחות היום
   - סטטוס שבועי (קצר)
   - אם יש תיקון — הסבר מה השתנה
   - לינק לדשבורד
6. אם היום הוא יום meal prep — תזכורת עם הפרטים

**פרומפט (תמצית):**
```
אתה סוכן התזונה של ערן. עשה צ'ק-אין יומי.
קרא את התפריט, הלוג והמאזן.
טפל בסטיות שלא טופלו, עדכן את הריפו, ושלח מייל עם ארוחות היום.
```

### רוטין 3: סיכום שבועי (Weekly Summary)

**טריגר:** Schedule — כל יום שישי בשעה 18:00

**מה עושה:**
1. קורא את `balance.json` — סיכום השבוע
2. מחשב: עמידה ביעדים, ממוצע יומי, ימים עם סטיות
3. כותב סיכום ל-`log.json`
4. שולח מייל סיכום שבועי לערן:
   - עמדת ביעד? כן/לא
   - ממוצע קלוריות יומי
   - ממוצע חלבון יומי
   - מספר סטיות
   - מה עבד טוב, מה פחות
5. דוחף לריפו

## ערוץ תקשורת — שיחה בקלוד

### מתי ערן משתמש בשיחה

- דיווח סטייה: "אכלתי פיצה בצהריים"
- שאלה מהירה: "מה אני אוכל עכשיו?"
- עדכון משקל: "אני שוקל 86"
- בקשת שינוי: "אני לא רוצה עוף השבוע"

### מה קורה בשיחה

1. ערן כותב את הדיווח
2. קלוד (בשיחה) מעדכן את `log.json` בריפו (push)
3. אם מדובר בסטייה — קלוד גם מעדכן את `balance.json`
4. הרוטין היומי הבא (08:00 למחרת) רואה את הלוג ומטפל בתיקון התפריט

### חלופה — עדכון מיידי

אם ערן רוצה תיקון מיידי (לא לחכות לרוטין הבא):
- קלוד בשיחה מעדכן גם את `menu.json` ישירות
- דוחף לריפו
- הדשבורד מתעדכן מיד

## סיכום

המערכת בנויה כך שכל רכיב עצמאי:

- **הריפו** עובד גם בלי רוטינס (אפשר לעדכן ידנית)
- **הדשבורד** עובד גם בלי רוטינס (מציג מה שיש בריפו)
- **הרוטינס** הם שכבת האוטומציה שמפעילה הכל בלי מגע

ככה אם רוטינס ישתנו, נשברו, או הגיעו למגבלה — המערכת עדיין עובדת, רק דורשת יותר עבודה ידנית.
